# CakeWorld Bot Manager v3

Веб-дашборд для управления Minecraft-ботами на сервере **mc.cakeworld.pw**.  
Написан на Node.js + Express + Socket.IO + mineflayer.

---

## Быстрый старт

```bash
# 1. Установить Node.js 18+ (https://nodejs.org)
# 2. Запустить — дважды кликнуть start.bat
```

`start.bat` сам установит зависимости при первом запуске, затем откроет сервер на **http://localhost:3000**.  
При падении сервер перезапускается автоматически через 3 секунды.

---

## Запуск вручную

```bash
npm install
node --expose-gc --max-old-space-size=512 server.js
```

Флаг `--expose-gc` включает ручную сборку мусора (нужен для экономии RAM).  
`--max-old-space-size=512` ограничивает кучу Node.js до 512 МБ.

## Архитектура

```
bot_dashboard/
├── server.js        — Node.js сервер: боты, API, Socket.IO
├── public/
│   ├── index.html   — главный дашборд
│   └── bot.html     — страница одного бота
├── package.json
└── start.bat        — запуск на Windows
```

**Стек:**
- `express` — HTTP сервер и REST API
- `socket.io` — real-time обновления в браузере (логи, статусы, рилики)
- `mineflayer` — Minecraft-клиент (подключение ботов)

## Функции

### Создание ботов

Кнопка **+ Создать ботов** открывает модальное окно с настройками:

| Поле | Описание |
|------|----------|
| Ник (base) | Базовое имя. Если создаёшь 5 ботов с ником `bot`, получишь `bot_1`, `bot_2`, ... |
| Пароль | Пароль для /register и /login. По умолчанию = ник |
| Сервер / Порт | Хост и порт Minecraft сервера |
| Версия | Версия протокола (например `1.20.1`) |
| Количество | Сколько ботов создать за раз (до 100) |
| Размер волны | Сколько ботов запускать одновременно (защита от флуда) |
| Задержка между волнами | Пауза между партиями ботов (мс) |
| Мир грифа | Номер мира (1–5) куда бот телепортируется через меню |
| Авто-авторизация | Автоматически отвечать на /register и /login при входе |
| Авто-навигация | Автоматически проходить через меню выбора сервера |
| Авто-AFK | Автоматически идти на /warp afk после входа на гриф |
| Авто-награды | Автоматически забирать /free по расписанию |
| Авто-реконнект | Переподключаться при дисконнекте с экспоненциальной задержкой |
| KitFarm | Автоматически забирать /kit и складывать в сундук |

---

### Авто-авторизация

Бот следит за сообщениями сервера:
- Если сервер просит `/register` — бот регистрируется
- Если просит `/login` — входит
- Если уже авторизован (приветственное сообщение) — переходит к навигации

Повторные попытки идут через 4 секунды с защитой от зацикливания.

---

### Авто-навигация (меню сервера)

После авторизации бот:
1. Берёт предмет из хотбара (компас)
2. Правым кликом открывает меню выбора сервера
3. Кликает слот 22 — **Гриферское выживание**
4. В следующем меню кликает нужный **мир грифа** (настраивается)
5. После входа на гриф запускает AFK, награды, KitFarm

---

### Авто-AFK

После входа на гриф бот:
1. Отправляет `/warp afk`
2. Ждёт телепорт (событие `forcedMove`)
3. Включает физику и идёт вперёд 3.2 секунды
4. Стадия меняется на **AFK**

Каждые 20 секунд бот слегка поворачивает голову (на 0.001 рад) — это предотвращает кик сервера за неактивность.

---

### Авто-награды (/free)

Бот забирает бесплатные награды по времени онлайна:

```
15, 45, 75, 120, 180, 300, 420, 520, 720, 1440 минут
```

При каждом сроке:
1. Бот отправляет `/free`
2. Ждёт открытия кастомного GUI-меню (до 7 секунд)
3. Кликает по всем активным слотам (не glass pane, не air) с паузой 600 мс
4. Закрывает меню
5. Планирует следующую награду

---

### KitFarm (автоматический сбор китов)

Бот-фармер автоматически:
1. Ждёт 2 секунды после входа на гриф
2. Телепортируется через `/warp mamka`
3. Берёт все киты из конфига (до 3 попыток при кулдауне)
4. Ищет ближайший сундук (радиус 10 блоков)
5. Открывает сундук и перекладывает еду
6. Выходит с сервера (`mc.quit()`) — бот переподключится и повторит

**Овнер-бот** (kitFarmRegionOwner): особая роль, сначала прописывает всех фармеров в регион, затем запускает их подключение.

---

### Авто-реконнект

При дисконнекте бот переподключается с экспоненциальной задержкой:

```
Попытка 1 → 5с
Попытка 2 → 10с
Попытка 3 → 20с
Попытка 4 → 40с
Попытка 5+ → 60с (максимум)
```

**Специальные случаи:**
- Кик с сообщением **"не прошел проверку / список зараженных"** — реконнект останавливается (IP заблокирован, нет смысла долбиться)
- Кик с **"ban/заблокир"** — бот помечается как забанен навсегда

---

### Рилики (Relics)

Бот парсит скорборд сервера и отслеживает количество риликов у каждого бота в реальном времени.

- Отображается в сайдбаре рядом с ником бота
- Обновляется при любом изменении скорборда
- Кнопка **🏆 Лидерборд риликов** открывает `/relics.html` с таблицей всех ботов по убыванию

---

### Магазин (/shop)

Кнопка **🛒 Магазин** в панели бота открывает интерактивный браузер магазина:

- Отправляет `/shop` и парсит открывшееся GUI-окно
- Показывает сетку 9×N с предметами, названиями, описанием (лором) и ценой
- **Зелёные карточки (▶)** — разделы/категории, клик переходит внутрь
- **Синие карточки (💎 цена)** — товары, клик покупает
- Кнопка **← Назад** (нижний-левый слот) возвращает в предыдущее меню
- Цена определяется из лора ИЛИ из количества предмета (кастомные магазины используют `count` как цену)
- Иконки определяются по русскому/английскому названию предмета

**🔍 Сканировать всё** — бот автоматически обходит все разделы магазина, собирает все товары с ценами и выводит итоговую таблицу (сортировка по цене).

---

### Консоль всех ботов (Broadcast)

Кнопка **📡 Все боты** открывает глобальную консоль:

- Показывает логи **всех** активных ботов в одном потоке (с именем бота)
- Поле ввода отправляет команду/сообщение **одновременно всем** онлайн-ботам
- Счётчик сообщений в шапке

---

### Реконнект всех

Кнопка **↺ Реконнект всех** — перезапускает подключение для каждого бота с паузой 300 мс между ботами.

---

### ПАСХА и Кейс

| Кнопка | Действие |
|--------|----------|
| 🐣 ПАСХА | Отправляет `/pasxa` выбранному или всем ботам |
| 📦 Кейс | Отправляет команду открытия кейса выбранному или всем ботам |

---

### Экспорт / Импорт ботов

**📤 Экспорт** — скачивает `bots.txt` со всеми ботами:
```
# CakeWorld Bot Manager
username1:password1
username2:password2
```

**📥 Импорт** — загружает ботов из TXT-файла:
- Формат: `username:password` или просто `username` (пароль = ник)
- Строки начинающиеся с `#` — комментарии, игнорируются
- Настройки сервера берутся из модала (автозаполняются из первого существующего бота)
- Поддержка до 500 ботов за раз с волновым запуском

## Оптимизация памяти

Каждый бот настроен на минимальное потребление RAM:

- `physicsEnabled: false` — физика выключена (не нужна для ботов-фармеров)
- `viewDistance: 'tiny'` — минимальная дистанция прогрузки чанков
- `checkTimeoutInterval: 60000` — редкая проверка таймаутов
- Каждые 30 секунд: выгрузка всех чанков + очистка кешированных сущностей + ручной вызов GC
- Лог каждого бота ограничен 60 записями
- Node.js запускается с `--max-old-space-size=512` и `--expose-gc`

## Socket.IO события

| Событие | Направление | Описание |
|---------|-------------|----------|
| `stats` | server→client | Общая статистика (онлайн/офлайн/бан/RAM/CPU/Heap) |
| `stats:live` | server→client | Live-обновление каждые 3 секунды |
| `bot:created` | server→client | Новый бот добавлен |
| `bot:removed` | server→client | Бот удалён |
| `bot:stage` | server→client | Изменился статус/стадия бота |
| `bot:log` | server→client | Новая строка в логе бота |
| `bot:riliky` | server→client | Обновились рилики бота |
| `relics:summary` | server→client | Итоговая таблица риликов |
| `shop:scan` | server→client | Результат сканирования магазина |
