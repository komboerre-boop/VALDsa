## Установка и запуск

### Windows (через батник)

```
Дважды кликнуть start.bat
```

Батник автоматически:
1. Проверяет наличие Node.js
2. Устанавливает зависимости при первом запуске (`npm install`)
3. Запускает сервер с оптимальными флагами
4. При падении — перезапускает через 3 секунды

После запуска открой браузер: **http://localhost:3000**

### Windows (вручную)

```bash
cd bot_dashboard
npm install
node --expose-gc --max-old-space-size=512 server.js
```

### Linux / macOS

```bash
cd bot_dashboard
npm install
node --expose-gc --max-old-space-size=512 server.js
```

### Шаг 1 — Установить Node.js на сервере

```bash
# Подключись к серверу по SSH
ssh root@IP_СЕРВЕРА

# Установить Node.js 20 через NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs

# Проверить версию
node -v   # должно быть v20.x.x
npm -v
```

### Шаг 3 — Запустить через PM2 (процесс-менеджер)

PM2 держит приложение запущенным 24/7, перезапускает при падениях и показывает логи.

```bash
# Установить PM2 глобально
npm install -g pm2

# Запустить бота
cd /root/bot_dashboard
pm2 start server.js --name "bot-manager" -- --expose-gc --max-old-space-size=512

# Автозапуск при перезагрузке сервера
pm2 startup
pm2 save

# Полезные команды PM2:
pm2 status              # статус всех процессов
pm2 logs bot-manager    # логи в реальном времени
pm2 restart bot-manager # перезапустить
pm2 stop bot-manager    # остановить
pm2 delete bot-manager  # удалить из PM2
```

### Шаг 4 — Открыть порт и доступ из интернета

По умолчанию сервер слушает порт **3000**. Чтобы зайти с другого компьютера:

```bash
# Открыть порт в файрволле (Ubuntu/Debian)

# Проверить что сервер слушает
ss -tlnp | grep 3000
```

Дашборд будет доступен по адресу: **http://IP_СЕРВЕРА:3000**

### Шаг 5 (опционально) — Домен + Nginx + HTTPS

Если хочешь доступ по красивому адресу `https://mybot.example.com`:

```bash
# Установить Nginx
apt-get install -y nginx certbot python3-certbot-nginx

# Создать конфиг для сайта
nano /etc/nginx/sites-available/botmanager
```

Вставь конфиг:

```nginx
server {
    listen 80;
    server_name mybot.example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;

        # Обязательно для Socket.IO
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }
}
```

```bash
# Активировать конфиг
ln -s /etc/nginx/sites-available/botmanager /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx

# Получить SSL сертификат (бесплатный Let's Encrypt)
certbot --nginx -d mybot.example.com
```

После этого дашборд доступен по **https://mybot.example.com**

### Создание ботов

Кнопка **+ Создать ботов** открывает форму с полными настройками.

**Поля:**

| Поле | Тип | По умолчанию | Описание |
|------|-----|-------------|----------|
| Ник (base) | строка | — | Базовый никнейм. При создании нескольких ботов добавляется суффикс `_1`, `_2`, ... Максимум 16 символов (ограничение Minecraft) |
| Пароль | строка | = ник | Пароль для /register и /login. Если не задан — используется ник |
| Сервер | строка | — | Хост Minecraft сервера (например `mc.cakeworld.pw`) |
| Порт | число | 25565 | TCP-порт сервера |
| Версия | строка | 1.20.1 | Версия Minecraft-протокола |
| Количество | 1–100 | 1 | Сколько ботов создать |
| Размер волны | 1–50 | 10 | Сколько ботов подключать одновременно |
| Задержка волны | 0–60000 мс | 4000 | Пауза между волнами |
| Мир грифа | 1–5 | 1 | Номер мира в меню "Выбор мира грифа" |
| Авто-авторизация | bool | ✅ | Автоматически /register и /login |
| Авто-навигация | bool | ✅ | Автоматически проходить меню выбора сервера |
| Авто-AFK | bool | ✅ | Автоматически идти на /warp afk |
| Авто-награды | bool | ✅ | Забирать /free по расписанию |
| Авто-реконнект | bool | ✅ | Переподключаться при дисконнекте |
| KitFarm | bool | ❌ | Автоматически фармить /kit |

**Волновая система запуска** защищает от одновременного подключения сотни ботов. Например, 50 ботов с волной=10 и задержкой=4с запустятся так:
```
0с    → боты 1-10 подключаются
4.2с  → боты 11-20
8.4с  → боты 21-30
12.6с → боты 31-40
16.8с → боты 41-50
```

### Авто-авторизация

Бот анализирует каждое сообщение от сервера и реагирует:

```
Сервер: "Введите /register <пароль>"  →  Бот: /register mypassword
Сервер: "Введите /login <пароль>"     →  Бот: /login mypassword
Сервер: "Добро пожаловать!"           →  Бот помечается авторизованным
Сервер: "Вы уже зарегистрированы"     →  Бот: /login mypassword
```

**Защита от зацикливания:** между попытками авторизации — пауза 4 секунды. После успешной авторизации счётчики попыток сбрасываются.

**Задержка перед отправкой пароля:** 1.4 секунды — чтобы сервер успел обработать предыдущий пакет.

---

### Авто-навигация (меню выбора сервера)

После авторизации бот берёт предмет из первого слота хотбара и открывает меню:

```
1. Берёт предмет из хотбара (компас или другой предмет-меню)
2. Нажимает ПКМ (activateItem) — открывается GUI
3. Видит меню "Выбор сервера" → кликает слот 22 (Гриферское выживание)
4. Видит меню "Выбор мира грифа" → кликает нужный мир (слоты 11-15)
5. Бот на гриф-сервере — запускает AFK, /free, KitFarm
```

Если открылось неизвестное меню — фолбэк: пробуем стандартные слоты.  
Если меню не открылось вообще — навигация повторяется через некоторое время.

---

### Авто-AFK

```
1. Бот отправляет /warp afk
2. Ждёт событие forcedMove (телепорт от сервера) — до 12 секунд
3. Включает физику (physicsEnabled = true)
4. Идёт вперёд + спринт 3.2 секунды → попадает в пул
5. Останавливается, физика выключается (physicsEnabled = false)
6. Статус → AFK
```

**Анти-таймаут:** каждые 20 секунд бот поворачивает голову на 0.001 радиан — это сигнализирует серверу об активности без реальных действий. Предотвращает кик "client timed out after 30s".

---

### Авто-награды (/free)

Сервер cakeworld выдаёт бесплатные награды за время онлайна. Расписание:

```
15 мин → первая награда
45 мин → вторая
75 мин → третья
...
1440 мин (24 часа) → последняя (10-я)
```

**Как это работает:**
```
1. Бот вошёл на гриф → запускается таймер
2. За N минут до нужного времени планируется setTimeout
3. В нужный момент: бот отправляет /free
4. Сервер открывает кастомное GUI-меню (10 слотов)
5. Бот кликает все активные слоты с паузой 600 мс
6. Меню закрывается, планируется следующая награда
7. Если меню не открылось за 7 секунд — награда пропускается, планируется следующая
```

При реконнекте таймер пересчитывается от момента входа на гриф.

### KitFarm

Автоматизация сбора ресурсов через /kit команды.

**Роли ботов:**

**Обычный фармер** (`kitFarm: true`):
```
1. Входит на гриф
2. Ждёт сигнала от овнер-бота (kitFarmReady = true)
3. Ждёт 2 секунды
4. /warp mamka → ждёт телепорт
5. Для каждого /kit: до 3 попыток при кулдауне
6. Находит ближайший сундук (радиус 10 блоков)
7. Открывает сундук → перекладывает еду
8. /quit → реконнект → повтор
```

**Овнер-бот** (`kitFarmRegionOwner: "никбота"`):
```
1. Входит на гриф
2. Прописывает всех других фармеров-ботов в регион (/rg addmember ...)
3. Устанавливает kitFarmReady = true для каждого фармера
4. Запускает их подключение
```

**Кулдаун /kit:** если сервер ответил "кулдаун N секунд" — бот ждёт ровно это время + 0.5 секунды, затем повторяет.

### Авто-реконнект

```
Дисконнект
    ↓
Определяем причину:
  "IP заблокирован" → СТОП (нет смысла переподключаться)
  "ban/заблокирован" → помечаем BANNED навсегда
  всё остальное → планируем реконнект
    ↓
Задержка: 5с → 10с → 20с → 40с → 60с → 60с → ...
    ↓
Переподключение
```

Задержка рассчитывается как `min(60с, 5с × 2^(попытка-1))`.

**IP-блокировка** (`"не прошел проверку / список зараженных"`):
- Реконнект останавливается
- В логе появляется сообщение "IP заблокирован сервером (реконнект остановлен)"
- Бот остаётся в статусе Offline (не Banned) — можно вручную переподключить когда IP разблокируют

### Рилики (Relics)

Рилики — внутриигровая валюта сервера, отображается на скорборде.

**Парсинг:**
1. Бот следит за событиями скорборда (`scoreboardCreated`, `scoreUpdated`, и др.)
2. При каждом изменении анализирует все строки скорборда через регулярное выражение
3. Ищет строки содержащие слова: `рилл`, `рилик`, `relic`
4. Извлекает число после этих слов

**Отображение:**
- В сайдбаре рядом с никнеймом каждого бота
- В строке статуса в главной панели
- На странице **/relics.html** — полная таблица со всеми ботами, сортировка по убыванию

### Магазин (/shop)

Полноценный браузер внутриигрового магазина.

**Открытие:**
```
Нажать кнопку 🛒 Магазин → бот отправляет /shop
Сервер открывает GUI-окно → бот парсит все слоты
Дашборд показывает сетку предметов
```

**Парсинг слотов:**
- Название предмета извлекается из `displayName` или `nbt.display.Name`
- Лор (описание) — из `nbt.display.Lore`
- Цена ищется в лоре по паттернам: `"цена: 50"`, `"100 риликов"`, `"💎 25"` и т.д.
- Если цена не найдена в лоре — берётся `count` предмета (многие кастомные магазины используют количество как цену)
- Декоративные стеклянные панели фильтруются автоматически
- Правый-нижний слот (счётчик валюты) игнорируется

**Типы слотов:**
- 🟢 Зелёная рамка + `▶` — раздел/категория, клик переходит внутрь
- 🔵 Синяя рамка + `💎 цена` — товар, клик покупает
- `←` (нижний-левый) — кнопка назад, возвращает в предыдущее меню

**Кнопка 🔍 Сканировать всё:**
```
1. Открывает /shop (корневое меню)
2. Для каждого не-декоративного слота:
   а. Кликает слот
   б. Если открылось новое окно → это раздел, записывает все товары внутри
   в. Если новое окно не открылось → это товар, записывает его
   г. Закрывает раздел, переоткрывает /shop
3. Показывает таблицу всех товаров с ценами (сортировка по цене)
```

### Консоль всех ботов (Broadcast)

Открывается кнопкой **📡 Все боты** — выдвижная панель снизу экрана.

**Лог:**
- Все логи от всех ботов в реальном времени
- Каждая строка: `[время] [ник бота] сообщение`
- Цветовая кодировка по типу (ошибка — красный, успех — зелёный, и т.д.)
- Счётчик сообщений в шапке

**Отправка команды:**
```
Ввести текст в поле → Enter или "Отправить всем"
→ Команда отправляется через chatSafe всем онлайн-ботам
→ Между отправками каждому боту — 50 мс задержка
```

`chatSafe` — очередь чата с минимальной паузой 600 мс между командами одного бота (защита от флуд-кика).

### Экспорт / Импорт ботов

**Экспорт (📤):**
- Генерирует `bots.txt` в браузере (без запроса к серверу)
- Формат: `username:password`, по одному боту на строку
- Первая строка — комментарий `# CakeWorld Bot Manager`

**Импорт (📥):**
```
1. Нажать 📥 Импорт
2. В модале указать сервер/порт/версию
   (автозаполняется из первого существующего бота)
3. Опционально — пароль по умолчанию (если в файле только ники)
4. Выбрать .txt файл
5. Нажать "Импортировать"
```

**Поддерживаемые форматы строк:**
```
username:password      — ник и пароль через двоеточие
username               — пароль = ник (или из поля "пароль по умолчанию")
# комментарий          — строки с # игнорируются
                        — пустые строки игнорируются
```

Боты запускаются с той же волновой системой что и при ручном создании. Максимум 500 ботов за один импорт.

### Оптимизация памяти

Каждый бот настроен на минимальное потребление RAM:

```js
physicsEnabled: false      // Выключена физика — основная экономия
viewDistance: 'tiny'       // Минимальная прогрузка чанков
checkTimeoutInterval: 60000 // Редкая проверка таймаутов
```

Каждые **30 секунд** для каждого бота:
1. `mc.world.unloadAllChunks()` — выгрузка всех чанков
2. Очистка кеша сущностей (оставляем только самого бота)
3. `gc()` — принудительный вызов сборщика мусора V8

Дополнительно:
- Максимум **60 записей** в логе каждого бота (старые удаляются)
- Node.js запускается с `--max-old-space-size=512` (лимит кучи)
- `--expose-gc` открывает доступ к ручному GC

**Примерное потребление:**
- 1 бот — ~15–25 МБ RAM
- 10 ботов — ~80–150 МБ
- 50 ботов — ~400–700 МБ

## REST API

### Боты

```http
GET /api/bots
```
Возвращает массив всех ботов с конфигом, статусом, логами, риликами.

```http
POST /api/bots
Content-Type: application/json

{
  "host": "mc.cakeworld.pw",
  "username": "mybot",
  "authPassword": "password",
  "port": "25565",
  "version": "1.20.1",
  "autoAuth": true,
  "autoNav": true,
  "autoAfk": true,
  "autoFree": true,
  "autoReconnect": true,
  "griefWorld": 1
}
```

```http
POST /api/bots/batch
Content-Type: application/json

{
  "host": "mc.cakeworld.pw",
  "base": "bot",
  "count": 20,
  "waveSize": 10,
  "waveDelay": 4000,
  "authPassword": "password",
  "version": "1.20.1"
}
```

```http
POST /api/bots/import
Content-Type: application/json

{
  "host": "mc.cakeworld.pw",
  "port": "25565",
  "version": "1.20.1",
  "bots": [
    { "username": "bot1", "password": "pass1" },
    { "username": "bot2", "password": "pass2" }
  ]
}
```

```http
GET /api/bots/export
```
Скачивает `bots.txt` со всеми ботами.

```http
DELETE /api/bots/:id
```
Удаляет бота и разрывает соединение.

### Управление

```http
POST /api/bots/:id/reconnect       — реконнект одного бота
POST /api/bots/reconnect/all       — реконнект всех ботов
POST /api/bots/broadcast           — отправить сообщение всем
  { "message": "/warp afk" }
POST /api/bots/:id/afk             — принудительно идти в AFK
POST /api/bots/:id/pasxa           — /pasxa для одного бота
POST /api/bots/pasxa/all           — /pasxa для всех
POST /api/bots/:id/case            — кейс для одного бота
POST /api/bots/case/all            — кейс для всех
```

### Магазин

```http
POST /api/bots/:id/shop/open       — открыть /shop
POST /api/bots/:id/shop/click      — кликнуть слот
  { "slot": 13 }
POST /api/bots/:id/shop/close      — закрыть магазин
POST /api/bots/:id/shop/scan       — запустить полное сканирование
```

## Socket.IO события

```
server → client:

stats              — {online, offline, banned, ram, cpu, heap, bots:[...]}
stats:live         — то же, раз в 3 секунды
bot:created        — {id, config, status, stage, logs}
bot:removed        — {id}
bot:stage          — {id, stage, status}
bot:log            — {id, type, text, ts}
                     type: info | warn | error | success | system | action | ban
bot:riliky         — {id, riliky}
relics:summary     — [{id, username, relics}]
shop:scan          — {id, result: {sections, items}}
```

## Переменные окружения

```bash
PORT=3000    # порт HTTP сервера (по умолчанию 3000)
```

```bash

```
